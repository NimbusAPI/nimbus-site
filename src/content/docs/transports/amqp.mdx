## Overview

The Nimbus AMQP transport provides connectivity to any AMQP 1.0 compliant message broker. While optimized for **ActiveMQ Artemis**, it works with any broker supporting the AMQP protocol, including RabbitMQ (with AMQP plugin), Apache Qpid, and others.

AMQP (Advanced Message Queuing Protocol) is an open standard protocol for message-oriented middleware, providing interoperability between different platforms and vendors.

## Key Benefits

- **Open Standard**: Not tied to any vendor
- **Full Control**: Self-hosted infrastructure
- **Protocol Flexibility**: AMQP 1.0 compatible
- **Feature Rich**: Transactions, acknowledgments, routing
- **Cross-Platform**: Works with multiple brokers
- **No Cloud Dependency**: Run on-premises or private cloud
- **ActiveMQ Artemis Optimized**: Best performance with Artemis

## When to Use AMQP

**Great for**:
- On-premises deployments
- Multi-cloud or hybrid cloud scenarios
- Organizations with existing AMQP infrastructure
- Requirements for open standards
- Need for full infrastructure control
- ActiveMQ Artemis users

**Consider alternatives if**:
- You want fully managed cloud service (use Azure Service Bus)
- You need maximum performance (use Redis)
- You want minimal setup complexity (use Redis or In-Process for testing)

## Installation

Install the AMQP transport package:

```bash
dotnet add package Nimbus.Transports.AMQP
```

## ActiveMQ Artemis Setup

### Install ActiveMQ Artemis

**Using Docker** (fastest):
```bash
docker run -d \
  --name artemis \
  -p 5672:5672 \
  -p 8161:8161 \
  -e ARTEMIS_USERNAME=admin \
  -e ARTEMIS_PASSWORD=admin \
  apache/activemq-artemis:latest
```

Or using Docker Compose:
```yaml
services:
  artemis:
    image: apache/activemq-artemis:latest
    ports:
      - "5672:5672"    # AMQP
      - "8161:8161"    # Web Console
    environment:
      - ARTEMIS_USERNAME=admin
      - ARTEMIS_PASSWORD=admin
```

**Manual installation**:
```bash
# Download from https://activemq.apache.org/components/artemis/
cd apache-artemis-2.x.x
bin/artemis create mybroker
cd mybroker
bin/artemis run
```

**Access web console**: http://localhost:8161 (admin/admin)

## Quick Start

### Basic Configuration

```csharp
using Nimbus;
using Nimbus.Configuration;
using Nimbus.Transports.AMQP;

var transport = new AMQPTransportConfiguration()
    .WithBrokerUri("amqp://localhost:5672")
    .WithCredentials("admin", "admin")
    .WithConnectionPoolSize(10);

var bus = new BusBuilder()
    .Configure()
    .WithNames("OrderService", Environment.MachineName)
    .WithTransport(transport)
    .WithTypesFrom(typeProvider)
    .WithAutofacDefaults(container)
    .Build();

await bus.Start();
```

### Send a Message

```csharp
public class PlaceOrderCommand : IBusCommand
{
    public string OrderId { get; set; }
    public decimal Amount { get; set; }
}

await bus.Send(new PlaceOrderCommand
{
    OrderId = "ORDER-123",
    Amount = 99.99m
});
```

## Connection Configuration

### WithBrokerUri(string uri)

Sets the AMQP broker URI (required):

```csharp
var transport = new AMQPTransportConfiguration()
    .WithBrokerUri("amqp://localhost:5672");
```

### WithCredentials(string username, string password)

Sets authentication credentials (optional):

```csharp
var transport = new AMQPTransportConfiguration()
    .WithBrokerUri("amqp://localhost:5672")
    .WithCredentials("admin", "admin");
```

### WithConnectionPoolSize(int poolSize)

Sets the connection pool size (default: 10):

```csharp
var transport = new AMQPTransportConfiguration()
    .WithBrokerUri("amqp://localhost:5672")
    .WithCredentials("admin", "admin")
    .WithConnectionPoolSize(20);
```

**Recommendations:**
- Low traffic: 5-10 connections
- Medium traffic: 10-20 connections
- High traffic: 20-50 connections

### WithFailover(params string[] uris)

Enables failover to multiple brokers for high availability:

```csharp
var transport = new AMQPTransportConfiguration()
    .WithFailover("amqp://broker1:5672", "amqp://broker2:5672")
    .WithCredentials("admin", "admin");
```

### WithClientId(string clientId)

Sets a client ID for durable subscriptions (optional):

```csharp
var transport = new AMQPTransportConfiguration()
    .WithBrokerUri("amqp://localhost:5672")
    .WithCredentials("admin", "admin")
    .WithClientId("MyApp-Instance1");
```

### From Configuration

```csharp
// appsettings.json
{
  "Messaging": {
    "AMQP": {
      "BrokerUri": "amqp://localhost:5672",
      "Username": "admin",
      "Password": "admin",
      "ConnectionPoolSize": 10
    }
  }
}

// Program.cs
var amqpSettings = configuration.GetSection("Messaging:AMQP");
var transport = new AMQPTransportConfiguration()
    .WithBrokerUri(amqpSettings["BrokerUri"])
    .WithCredentials(amqpSettings["Username"], amqpSettings["Password"])
    .WithConnectionPoolSize(int.Parse(amqpSettings["ConnectionPoolSize"]));
```

## ActiveMQ Artemis Configuration

### broker.xml Configuration

Optimize Artemis for Nimbus:

```xml
<configuration>
  <core>
    <!-- Connection settings -->
    <acceptors>
      <acceptor name="amqp">tcp://0.0.0.0:5672?protocols=AMQP</acceptor>
    </acceptors>

    <!-- Address settings for auto-created queues -->
    <address-settings>
      <address-setting match="#">
        <!-- Message retention -->
        <max-size-bytes>10485760</max-size-bytes>
        <page-size-bytes>1048576</page-size-bytes>

        <!-- Redelivery -->
        <max-delivery-attempts>5</max-delivery-attempts>
        <redelivery-delay>5000</redelivery-delay>

        <!-- Dead letter address -->
        <dead-letter-address>DLQ</dead-letter-address>

        <!-- Expiry -->
        <expiry-address>ExpiryQueue</expiry-address>
        <expiry-delay>-1</expiry-delay>

        <!-- Auto-delete unused queues -->
        <auto-delete-queues>true</auto-delete-queues>
        <auto-delete-queues-delay>86400000</auto-delete-queues-delay>
      </address-setting>
    </address-settings>
  </core>
</configuration>
```

### Performance Tuning

```xml
<!-- For high throughput -->
<address-setting match="#">
  <max-size-bytes>104857600</max-size-bytes>  <!-- 100MB -->
  <page-size-bytes>10485760</page-size-bytes>  <!-- 10MB -->
  <max-delivery-attempts>3</max-delivery-attempts>

  <!-- Enable paging for large queues -->
  <address-full-policy>PAGE</address-full-policy>
</address-setting>

<!-- Journal settings for durability -->
<journal-type>ASYNCIO</journal-type>
<journal-sync-transactional>true</journal-sync-transactional>
<journal-sync-non-transactional>false</journal-sync-non-transactional>
```

## Performance Configuration

### Concurrency Settings

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithMaxConcurrentHandlers(20)      // Process up to 20 messages concurrently
    .WithPrefetchCount(100)              // Prefetch for better throughput
    .WithConnectionPoolSize(5);          // Connection pool size
```

### Session Management

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithSessionPoolSize(10)             // Session pool size
    .WithSessionTimeout(TimeSpan.FromMinutes(5));
```

### Acknowledgment Settings

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithAckMode(AcknowledgmentMode.AutoAcknowledge)  // or ClientAcknowledge
    .WithTransactional(false);                         // Enable for transactions
```

<Callout type="info">
**Acknowledgment Modes**:
- **AutoAcknowledge**: Messages acknowledged automatically after delivery (faster, less reliable)
- **ClientAcknowledge**: Manual acknowledgment after processing (slower, more reliable)
- **Transactional**: Full transactional support (slowest, most reliable)
</Callout>

## Features

### Message Persistence

All messages are persisted by default with ActiveMQ Artemis:

```csharp
// Messages are durable by default
await bus.Send(new PlaceOrderCommand { OrderId = "123" });

// Survives broker restarts
```

### Dead Letter Queue

Failed messages are automatically moved to a dead letter queue with the prefix `DLQ.`:

```
Original Queue: MyApp.MyCommand
Dead Letter Queue: DLQ.MyApp.MyCommand
```

Messages use client acknowledgment mode and are acknowledged after successful processing. Failed messages are redelivered according to the broker's redelivery policy and eventually moved to the DLQ:

```csharp
// Dead letter queues are created automatically
// Monitor via Artemis web console at http://localhost:8161
// Or browse programmatically using Artemis management API
```

### Message Filtering

Use selectors to filter messages:

```csharp
public class HighPriorityOrderHandler : IHandleCommand<PlaceOrderCommand>
{
    // Only receive high-priority orders
    [MessageSelector("Priority > 5")]
    public async Task Handle(PlaceOrderCommand command)
    {
        // Process high-priority order
    }
}
```

### Scheduled Delivery

ActiveMQ Artemis supports native scheduled delivery using `_AMQ_SCHED_DELIVERY`. Messages are held by the broker until the scheduled time:

```csharp
// Delay a command by 5 minutes
await bus.SendAfter(new MyCommand { Data = "test" }, TimeSpan.FromMinutes(5));

// Schedule at specific time
var deliveryTime = DateTimeOffset.UtcNow.AddHours(1);
await bus.SendAt(new MyCommand { Data = "test" }, deliveryTime);
```

<Callout type="warning">
**ActiveMQ Artemis Only**: Scheduled messages require ActiveMQ Artemis 2.x+ and use the `_AMQ_SCHED_DELIVERY` header. ActiveMQ Classic 5.x has limited support for scheduled messages.
</Callout>

### Transactions

Full transactional support:

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithTransactional(true);

// Handler with transaction
public class PlaceOrderHandler : IHandleCommand<PlaceOrderCommand>
{
    public async Task Handle(PlaceOrderCommand command)
    {
        using (var tx = new TransactionScope(TransactionScopeAsyncFlowOption.Enabled))
        {
            await _repository.CreateOrder(command);
            await _bus.Send(new ProcessPaymentCommand { OrderId = command.OrderId });

            tx.Complete();  // Commit both database and message operations
        }
    }
}
```

### Large Messages

ActiveMQ Artemis supports large messages out of the box:

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithLargeMessageThreshold(102400);  // 100KB threshold

// Messages larger than threshold are handled as large messages
await bus.Send(new ProcessFileCommand
{
    FileData = largeByteArray  // Can be several MB
});
```

## High Availability

### Master-Slave Configuration

Configure ActiveMQ Artemis for HA:

**Master broker.xml**:
```xml
<ha-policy>
  <replication>
    <master>
      <check-for-live-server>true</check-for-live-server>
      <cluster-name>my-cluster</cluster-name>
    </master>
  </replication>
</ha-policy>
```

**Slave broker.xml**:
```xml
<ha-policy>
  <replication>
    <slave>
      <allow-failback>true</allow-failback>
      <cluster-name>my-cluster</cluster-name>
    </slave>
  </replication>
</ha-policy>
```

**Client configuration with failover**:
```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString(
        "failover:(amqp://master:5672,amqp://slave:5672)" +
        "?maxReconnectAttempts=10" +
        "&initialReconnectDelay=1000"
    )
    .WithUsername("admin")
    .WithPassword("admin");
```

### Clustering

Set up network of brokers for load distribution:

```xml
<cluster-connections>
  <cluster-connection name="my-cluster">
    <connector-ref>artemis</connector-ref>
    <discovery-group-ref discovery-group-name="my-discovery-group"/>
    <max-hops>1</max-hops>
    <message-load-balancing>ON_DEMAND</message-load-balancing>
  </cluster-connection>
</cluster-connections>
```

### Health Checks

```csharp
public class AmqpHealthCheck : IHealthCheck
{
    private readonly IAmqpConnection _connection;

    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context,
        CancellationToken cancellationToken = default)
    {
        try
        {
            // Test connection
            if (_connection.IsOpen)
            {
                return HealthCheckResult.Healthy("AMQP connection is open");
            }
            else
            {
                return HealthCheckResult.Unhealthy("AMQP connection is closed");
            }
        }
        catch (Exception ex)
        {
            return HealthCheckResult.Unhealthy("Cannot connect to AMQP broker", ex);
        }
    }
}
```

### Queue Monitoring

```csharp
public class QueueMonitor
{
    private readonly IArtemisManagementClient _managementClient;

    public async Task<QueueStatistics> GetQueueStats(string queueName)
    {
        // Use Artemis management API
        var stats = await _managementClient.GetQueueStats(queueName);

        return new QueueStatistics
        {
            MessageCount = stats.MessageCount,
            ConsumerCount = stats.ConsumerCount,
            MessagesAdded = stats.MessagesAdded,
            MessagesAcknowledged = stats.MessagesAcknowledged
        };
    }
}
```

## Best Practices

### 1. Use Connection Pooling

```csharp
// ✅ GOOD: Single bus instance (singleton)
services.AddSingleton<IBus>(provider =>
{
    var bus = new BusBuilder()
        .Configure()
        .WithTransport(transport)
        .Build();
    bus.Start().Wait();
    return bus;
});
```

### 2. Configure Appropriate Timeouts

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithRequestTimeout(TimeSpan.FromSeconds(30))
    .WithIdleTimeout(TimeSpan.FromMinutes(5));
```

### 3. Enable Dead Letter Queue

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithMaxDeliveryAttempts(5)
    .WithDeadLetterQueueEnabled(true);
```

### 4. Monitor Queue Depths

Set up alerts for queue depth:
- Alert when queue depth > threshold for extended period
- Indicates consumers can't keep up with load

### 5. Use Persistent Messages

Messages are persistent by default, but verify configuration:

```xml
<!-- broker.xml -->
<journal-type>ASYNCIO</journal-type>
<journal-sync-transactional>true</journal-sync-transactional>
```

### 6. Implement Failover

Always configure failover for production:

```csharp
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("failover:(amqp://broker1:5672,amqp://broker2:5672)")
    .WithUsername("admin")
    .WithPassword("admin");
```

## Common Issues

### Issue: Connection refused

**Cause**: Broker not running or wrong port

**Solution**:
```bash
# Check broker is running
netstat -an | grep 5672

# Check Artemis logs
tail -f /path/to/artemis/log/artemis.log

# Verify connection string
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")  // Correct port?
```

### Issue: Authentication failed

**Cause**: Wrong credentials or user doesn't exist

**Solution**:
```bash
# Create user in Artemis
./artemis user add --user myuser --password mypass --role admin

# Or update credentials in code
.WithUsername("myuser")
.WithPassword("mypass")
```

### Issue: Messages going to DLQ

**Cause**: Handler throwing exceptions or processing timeout

**Solution**:
```csharp
// Increase delivery attempts
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithMaxDeliveryAttempts(10)
    .WithRequestTimeout(TimeSpan.FromMinutes(5));

// Make handler more resilient
public async Task Handle(Command command)
{
    try
    {
        await ProcessWithRetry(command);
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Failed to process");
        throw;  // Let AMQP handle retry
    }
}
```

### Issue: Slow message processing

**Cause**: Low concurrency or broker tuning

**Solution**:
```csharp
// Increase concurrency
var transport = new AmqpTransportConfiguration()
    .WithConnectionString("amqp://localhost:5672")
    .WithUsername("admin")
    .WithPassword("admin")
    .WithMaxConcurrentHandlers(50)
    .WithPrefetchCount(200);

// Tune Artemis broker.xml
<thread-pool-max-size>50</thread-pool-max-size>
<scheduled-thread-pool-max-size>10</scheduled-thread-pool-max-size>
```

## Limitations

### Namespace Cleaning

The `INamespaceCleanser` implementation is a no-op. Configure auto-deletion policies on the broker or use the ActiveMQ management console to clean up queues/topics during testing.

### Message Filtering

JMS selectors are not currently implemented. Filtering is performed at the handler level rather than at the broker level.

## Compatibility

- ✅ **ActiveMQ Artemis 2.x+** (Recommended - full feature support)
- ✅ **ActiveMQ Classic 5.12+** (Limited support - delayed delivery not fully supported)
- ✅ **.NET Standard 2.0+**
- ✅ **Apache NMS AMQP 2.2.0**

## Performance Considerations

### Connection Pool Size

The connection pool size determines how many concurrent operations can be performed. A larger pool allows more parallelism but consumes more broker resources.

### Session Management

Each send/receive operation creates a temporary session. Sessions are lightweight and don't need pooling.

### Message Acknowledgment

Messages use client acknowledgment mode. They are acknowledged after successful processing. Failed messages are redelivered according to the broker's redelivery policy.
